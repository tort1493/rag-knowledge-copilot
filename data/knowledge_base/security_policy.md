# Security Policy

Atlas encrypts customer data in transit using TLS 1.2 or higher and encrypts data at rest using AES-256.

Production access is limited by role-based access control and requires multi-factor authentication. Shared administrator accounts are prohibited.

Security incidents that involve suspected unauthorized data access must be reported to the security team within 15 minutes of discovery. The security team maintains the authoritative incident record.

Atlas does not permit employees to copy customer data into public generative AI tools unless the data has been formally classified as public by the data governance lead.
